# 검색 평가 정책

제도 찾기(사이드바 AI)의 검색 품질을 어떻게 재는지 정한다.

## 왜 이 문서가 있나

제도를 맞췄는지만 재다가 실제 문제를 놓쳤다.

`"지체상금률이 얼마인가요"`에 대해 제도(`지체상금`)는 정확히 골랐지만, 근거로
딸려온 것은 **1,881자짜리 제25조(지체상금 일반)** 였다. 정작 요율이 적힌 것은
386자짜리 **제75조**였는데, 글자 2-gram 겹침으로 재니 긴 조가 같은 점수를 받았다.
제도 적중률은 100%로 나오는데 답은 동문서답이 되는 상태였다.

**제도를 맞히는 것과 답에 필요한 내용을 가져오는 것은 다른 문제다.**

## 반드시 함께 재는 세 가지

| 지표 | 무엇을 재나 | 실패하면 생기는 일 |
|---|---|---|
| **제도 적중률** | 질문에 맞는 제도를 골랐나 | 엉뚱한 제도 페이지로 안내 |
| **근거 포함률** | 답에 필요한 내용이 검색 결과에 들어왔나 | 제도는 맞는데 동문서답 |
| **근거 대조 통과율** | 모델이 쓴 문장이 원문에 실재하나 | 환각 |

**제도 적중률만 보고 판단하지 않는다.** 세 지표를 함께 보고, 하나라도 나빠지면
개선이 아니다.

## 근거 포함률을 어떻게 재나

질문마다 **답이 성립하려면 반드시 있어야 하는 문구**(`evidence`)를 미리 적어둔다.
검색이 실제로 모델에 넘긴 텍스트에 그 문구가 있는지 문자열로 대조한다.

```json
{
  "q": "지체상금률이 얼마인가요",
  "ok": ["liquidated-damages"],
  "evidence": ["1천분의 0.5", "1천분의 0.75", "1천분의 1.25"]
}
```

- `evidence`는 **원문에 그대로 있는 문구**여야 한다. 요약하거나 바꿔 쓰면 대조가
  무의미해진다.
- 숫자·기한·요건처럼 **틀리면 실무에 손해가 나는 것**을 우선 넣는다.
- 공백은 무시하고 대조한다(원문 줄바꿈 위치가 판정을 좌우하면 안 된다).

근거 포함률 = (검색 결과에 있는 evidence 수) / (전체 evidence 수)

이 지표는 **LLM을 부르지 않는다.** 검색 단계만 재므로 공짜이고 빠르며, 모델을
바꿔도 결과가 흔들리지 않는다.

## 언제 재나

검색에 손대는 변경은 전부 — 프리필터, 검색 단위, 점수식, 후보 개수, 인덱스 필드.

```
node scripts/eval/retrieval.mjs
```

## 판정 기준

- **근거 포함률이 떨어지면 그 변경은 개선이 아니다.** 속도가 빨라졌거나 토큰이
  줄었어도 마찬가지다. 답에 필요한 내용을 안 가져오면 나머지는 의미가 없다.
- 제도 적중률과 근거 포함률이 상충하면 **근거 포함률을 우선한다.** 제도가 조금
  빗나가도 필요한 조문이 들어오면 답은 나오지만, 그 반대는 안 된다.

## 기록

측정값은 커밋 메시지에 남긴다. 숫자 없는 "개선했다"는 주장은 받지 않는다 —
이 프로젝트에서 실제로 여러 번, 재보지 않고 내린 판단이 틀렸다.
